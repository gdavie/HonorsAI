Detailed Description

The interface/abstract definition of a program - this can be specialised, e.g. for hill- climbing programs. T is the type of the features and registers. Sub classes must only give the constructors, although they can override other methods too if need be. They should also assign an IFitnessMeasure of some type (the right type whatever that is) to the member "fitnessMeasure" in their constructors.

void core.PLGP_Program.calcDist ( )

MUTATIONJ Uses final register values to calculate the responsibilities of instructions in the program. These are then used to stochastically influence which instructions are selected to be mutated.

main loop:

  1. select the next instruction IF EXON
  2. assign to it prob equal to average of values associated with destination register
  3. update destination register to have no associated values
  4. add dest register value before updating to both register operands IF INTRON
  5. assign average prob to instruction.

void core.PLGP_Program.markIntrons ( )

Sets the IsIntron flag to true for each instruction in the program if that instruction is a structural intron, false otherwise. Marking structural introns can massively speed up execution on long programs.
